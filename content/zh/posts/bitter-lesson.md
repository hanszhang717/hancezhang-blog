### 被误读的“AI 圣经”

在 AI 圈里，Rich Sutton 的 **“The Bitter Lesson” (苦涩的教训)** 已经成了某种圣经。

它的核心教义是：试图把人类知识硬塞给模型，都是短视的拐杖。历史证明，唯一长久有效的，就是把海量算力砸在通用算法上。

这导致了一种行业教条：**让模型自己搞定一切。** 你的产品越“Agentic”（自主），就显得越纯粹、越高级。

但在做产品的战壕里，情况完全不同。**Bitter Lesson 在科研上是对的，但在做产品时，你需要学会“作弊”。**

### 顺毛摸驴

给 LLM 太多的自由通常是一场灾难。正确的姿势是**“顺毛摸驴”**。

LLM 就像木头一样，有它的纹理。它对某些代码模式烂熟于心。**如果你顺着它的纹理（Grain）提要求，它是神；如果你逆着它的纹理来，它就是人工智障。**

强迫 LLM 在上下文中现学一套全新的架构，是对 Context Window 的自杀式攻击。我们衡量模型不该只看它能吞多少字，而要看它的 **"In-Context Capability"**——在不发疯的前提下，它到底能处理多复杂的逻辑。

这就是为什么像 **Base44** 这样的工具好用。它们**作弊**了。它们大量使用了人类预设的模板，只让 LLM 做填空题。

这“不纯粹”吗？是的。但这管用吗？太管用了。

做产品，**时间窗口就是一切。** 做一个注定会被 GPT-6 淘汰的过渡性产品完全没问题。**如果加上一点人类规则就能在今天解决用户的问题，那就加上。** 技术是手段，不是宗教。

### AI 编程的四个阶段

**1. AI 辅助编程 (自行车)**
Tab 补全，Copilot。简单，无害，锦上添花。

**2. AI 结对编程 (手动挡)**
Cursor Composer, Windsurf。非常强大，但你得会开车。你必须懂代码，才能把住方向盘。

**3. Vibe Coding (自动挡)**
Lovable, Bolt.new。这是**民主化**的阶段。门槛被拉低了，不会写代码的人也能上路了。
但这体验就像是在跟机器人玩“你画我猜”。虽然有时候很挫败，但它让很多人第一次体验到了**“推背感”**。

**4. 终局：完全自动驾驶**
这是未来。你给出一个模糊的想法，AI 端到端地搞定一切。不需要你 Review 代码，不需要你修 Bug。直接给你一个能跑的 App。

### 软件的“杰文斯悖论”

有人担心 AI 能写代码了，程序员就要饿死了。

我不这么看。看看摄影。
以前拍照需要暗房、药水和专业技能。现在，只需要毫秒级的计算摄影。手机里的 ISP 瞬间完成了过去需要几小时的后期工作。

**手机摄影杀死了摄影吗？没有。它引爆了市场。**
它创造了 Instagram, TikTok, 和整个视觉经济。

AI 对软件也会做同样的事。**当软件变得像 JPEG 图片一样廉价时，我们不会少做软件。我们会多做一百万倍的软件。**

### 从卖眼球到卖 API

Agent 的崛起是广告模式的丧钟。
如果用户都用 AI 总结播客、浏览网页，**没人会看广告。**

旧模式（注意力 -> 广告）正在死去。
新模式是 **服务 -> API**。

创作者不该再为了博眼球而优化；应该为了**被 Agent 调用**而优化。把你的内容、数据、逻辑封装成 API 收费。在 Agent 统治的世界里，这可能是唯一的活路。
